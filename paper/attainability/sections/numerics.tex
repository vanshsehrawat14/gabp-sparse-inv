\section{Exactness is not accuracy}
\label{sec:numerics}

Before claiming exactness buys learning, the paper rules out the mundane explanation
that it buys accuracy. It does not, and this is the hinge of the argument:
the one axis where an exact method should win, numerical precision, is where
it does not win, which forces the advantage in Sections~\ref{sec:maze}
and~\ref{sec:deq} to be a learning-dynamics effect -- the attainability gap
those sections measure -- rather than a numerical one.

\paragraph{No robust win in either direction.}
Scored on the pattern against a 64-bit oracle and swept over conditioning
(Table~\ref{tab:precision}), the selected inverse and a dense SPD solve at
equal precision stay within small constant factors of each other across
structures and $\kappa$: the error ratio ranges from $0.3$ to $2.0$ with no
consistent direction (at the highest-$\kappa$ grid cell the dense solve is in
fact the more accurate, by about $3\times$). An initial reading of roughly $2\times$
on trees, growing with $\kappa$, on ill-scaled random matrices turned out,
over enough seeds (16 for the ordering study), to be a ratio-of-medians
artifact, and most of the modest, heavy-tailed median gap is an
elimination-ordering effect: a dense Cholesky run in the kernel's own
leaf-first order is just as accurate. If exactness conferred an accuracy
edge, this is where it would appear; it is absent.

\paragraph{What exactness does give: a cheap global operator.}
The edge is not the value of the number; it is that the operator is exact,
global, and cheap. Cheapness is what makes the operator usable as a layer:
Figure~\ref{fig:scaling} shows forward-plus-backward cost scaling as
$\bigO(n)$, with the measured dense crossover on a block chain, and a
differentiable GMRF learning at linear cost where dense autograd is
$\bigO(N^3)$. An exact, whole-graph functional at a price that does not grow
with problem size or graph diameter is the combination the next two sections
exploit.
